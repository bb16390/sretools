# Kafka 日志收集器重构 - 验证清单

## 功能验证

- [ ] Kafka 适配器能成功连接到 Kafka 集群
- [ ] Kafka 适配器能订阅主题并消费消息
- [ ] 消费进度能持久化到本地文件
- [ ] 消费进度能定期上报到 Master 端
- [ ] 启动时能从本地或 Master 端恢复消费进度
- [ ] Master 端能接收和存储消费进度
- [ ] Master 端能返回消费进度给 Worker
- [ ] LogCollector 能使用 Kafka 适配器消费消息
- [ ] LogCollectorTask 能正常启动和运行
- [ ] 没有消息被重复消费（从上次的偏移量继续）

## 代码质量验证

- [ ] 代码遵循项目现有的风格和架构
- [ ] 新的 Kafka 适配器正确继承 AsyncBaseAdapter
- [ ] 所有新增的配置项有合理的默认值
- [ ] 错误处理完善，有适当的日志记录
- [ ] 代码有必要的注释和文档
- [ ] 不影响现有功能的正常运行

## 配置验证

- [ ] Kafka 连接参数完全可配置
- [ ] 消费进度上报间隔可配置
- [ ] 配置项有默认值，向后兼容
- [ ] 配置能从环境变量加载（如果需要）

## 集成验证

- [ ] Worker 与 Master 端的通信正常
- [ ] 消费进度的上报和获取功能正常
- [ ] 整体流程端到端测试通过
- [ ] 模拟故障场景下系统表现稳定
